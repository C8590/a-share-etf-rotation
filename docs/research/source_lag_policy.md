# Source Lag Policy

This policy defines how A-ETF-OPEN treats ETF price coverage gaps that remain after a controlled refresh attempt.

## Definition

`source_lag` means one or a small number of ETF symbols trail the expected market date while the broader ETF cache has moved further forward.

This is different from ordinary stale cache:

- `stale_cache`: local cache is old and a normal targeted refresh is expected to fix it.
- `source_lag`: a controlled refresh or source check indicates the provider path itself is stale, unavailable, or blocked.
- `market_wide_lag`: many symbols lag together; a controlled market-wide refresh may be appropriate.

## Governance Rule

Source lag is a blocker, not a pass condition.

A source-lag symbol must stay out of:

- production candidate pool
- factor score promotion
- ETF-GAP-008B
- any automatic trading or broker execution path

The blocker can explain QA failure, but it must not relax QA.

Recommended action:

`keep blocked; diagnose provider/source lag; do not run full-market refresh for this alone`

## When To Retry Refresh

Retry only when at least one of these is true:

- provider diagnostics show the source has advanced beyond the stale symbol cache date
- EastMoney proxy/network access is fixed
- a controlled targeted refresh is requested for the affected symbol
- the source-lag report changes from `provider_stale` or `proxy_blocked` to a refreshable state

## When To Wait Or Change Source

Wait or change provider path when:

- Sina still returns the old date
- EastMoney paths are unavailable
- a single symbol is stale while the broader market cache is fresher
- retrying the full market would create churn without addressing the blocker

## Reports

- `output/source_lag_report.csv`: per-symbol source lag evidence and blocker decision.
- `output/source_lag_summary.csv`: aggregate counts and next source-diagnosis action.

These reports are generated by:

```powershell
.\.venv\Scripts\python.exe main.py diagnose-source-lag
```

The command is read-only with respect to price cache and strategy outputs. It does not refresh ETF cache, index cache, backtest, strategy signal, or factor candidates.
