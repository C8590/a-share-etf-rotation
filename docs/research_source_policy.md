# 研究来源审查标准

本项目的策略结论必须以本项目自己的行情数据、模拟盘样本、回测和可复现结果为准。外部资料只能作为研究线索，不能直接作为买入、退出、阈值或风控公式的依据。

## 基本原则

1. 机构研报、媒体信息、公告解读和人工备注只能作为研究线索或风险上下文。
2. 外部资料不得直接改写日频动量策略、entry 规则、exit 规则或仓位公式。
3. 2024-09-24 后的 A 股样本必须单独分段验证。
4. 策略结论必须覆盖交易成本、回撤、样本外和复盘病例。
5. 旧版结果只能作为历史对照 / 旧版参考，不参与当前交易建议。

## 与 risk_warning 的边界

`risk_warning` 是 entry 前的最高优先级风险门控模块。它可以记录外部系统性风险事件，并在风险升高时让 entry 降速、冻结或进入人工复核。

它不能被写成新闻驱动策略：

- 不推荐新闻。
- 不预测哪个 ETF 会涨。
- 不把研报或媒体观点直接转换为买入信号。
- 不替代 market_state、日频动量策略或人工判断。
- 不阻断卖出、减仓、止损和退出等风险降低动作。

## 外部资料登记字段

任何外部资料进入研究记录前，必须登记：

1. 来源类型。
2. 利益动机。
3. 样本区间。
4. 是否跨 2024-09-24。
5. 是否有交易成本。
6. 是否有样本外验证。
7. 是否存在高回撤低收益问题。
8. 是否只作为风险上下文，而不是买入依据。

## 审查结论分级

- A 级：本项目数据可复现，覆盖交易成本，有样本外验证，且关键样本分段均单独检查。
- B 级：有可复现数据或公开方法，但样本外验证、成本、回撤检查不完整。
- C 级：机构研报、媒体观点、主观判断或不可复现结论，只能作为研究线索。
- D 级：存在明显利益冲突、样本选择偏差、无成本假设或高回撤低收益问题，不进入策略讨论。

## 策略隔离要求

外部资料不得直接修改：

- `signal/daily_signal.py`
- `strategy/`
- `risk_warning/scorer.py`
- `risk_warning/gate.py`
- `signal/trade_policy.py`

如需根据外部资料提出调整建议，必须先进入研究记录或 learning 复盘上下文，经过样本分段、模拟盘病例、历史对照和可复现验证后，再作为单独策略评审议题处理。
